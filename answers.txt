Q1: Why might we consider using Git?

Answer: So that we can switch back to any previous version if somehow we mess up our code.

Q3: What is the difference between Merging and Rebasing?

Answer: Merging combines branches with a merge commit and keeps history intact, while rebasing moves commits onto another branch to create a cleaner, linear history.
